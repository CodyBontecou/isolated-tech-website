## Overview
Implement production error tracking to catch issues before users report them. Currently errors only log to console.

## Goals
- Capture and aggregate frontend errors
- Capture backend/API errors in Cloudflare Workers
- Get alerts for new/recurring issues
- Have context (user, page, request) for debugging

## Options Considered
1. **Sentry** - Industry standard, has Cloudflare Workers SDK
2. **LogRocket** - Session replay + errors, but heavier
3. **Cloudflare Logpush** - Native but less feature-rich
4. **Highlight.io** - Open source alternative

## Recommended: Sentry
- Best Cloudflare Workers support via `@sentry/cloudflare`
- Free tier sufficient for indie scale
- Source maps support for minified code
