## Overview
Offer a "Get all apps" bundle at a discounted price for superfans. Increases average order value and customer lifetime value.

## Goals
- Increase revenue per customer
- Reward loyal/enthusiastic customers
- Simplify purchasing for "I want everything" users
- Future-proof: bundle includes future apps

## Pricing Models

### Option 1: Fixed Bundle Price
- "All Apps Bundle" for $X (e.g., $49)
- Includes all current + future apps
- Simple to understand

### Option 2: Dynamic Discount
- Sum of all minimum prices, apply X% discount
- E.g., if individual apps total $80, bundle is $60 (25% off)
- Scales automatically as catalog grows

### Option 3: Tiered Bundles
- "macOS Bundle" - all macOS apps
- "iOS Bundle" - all iOS apps  
- "Everything Bundle" - all apps

## Recommendation
Start with **Option 1** (Fixed Bundle Price) for simplicity.

## Database Schema

### New Table: `bundles`
```sql
CREATE TABLE bundles (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,              -- "All Apps Bundle"
  slug TEXT UNIQUE NOT NULL,       -- "all-apps"
  description TEXT,
  price_cents INTEGER NOT NULL,    -- Fixed price
  includes_future_apps INTEGER DEFAULT 1,
  is_active INTEGER DEFAULT 1,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

### New Table: `bundle_apps` (for explicit bundles)
```sql
CREATE TABLE bundle_apps (
  bundle_id TEXT NOT NULL,
  app_id TEXT NOT NULL,
  PRIMARY KEY (bundle_id, app_id),
  FOREIGN KEY (bundle_id) REFERENCES bundles(id),
  FOREIGN KEY (app_id) REFERENCES apps(id)
);
```

### Modify `purchases` Table
```sql
ALTER TABLE purchases ADD COLUMN bundle_id TEXT REFERENCES bundles(id);
```

### 2. Checkout Flow
- Similar to app checkout but for bundle
- Creates purchase records for all included apps
- Marks as bundle purchase

### 3. Dashboard Display
- Show "Bundle" badge on purchased apps
- "You own the All Apps Bundle" message

## Implementation

### Checkout Logic
```typescript
// app/api/checkout/bundle/route.ts
export async function POST(request: Request) {
  const { bundleId, priceCents } = await request.json();
  
  const bundle = await getBundle(bundleId);
  const apps = await getBundleApps(bundleId);
  
  // Check if user already owns all apps
  const owned = await getUserOwnedApps(userId);
  const newApps = apps.filter(a => !owned.includes(a.id));
  
  if (newApps.length === 0) {
    return Response.json({ error: "You already own all bundled apps" });
  }
  
  // Create Stripe session
  const session = await stripe.checkout.sessions.create({
    line_items: [{
      price_data: {
        currency: 'usd',
        unit_amount: priceCents,
        product_data: { name: bundle.name },
      },
      quantity: 1,
    }],
    metadata: {
      bundle_id: bundleId,
      user_id: userId,
      app_ids: apps.map(a => a.id).join(','),
    },
  });
}
```

### Webhook Handler
```typescript
// On checkout.session.completed
if (metadata.bundle_id) {
  const appIds = metadata.app_ids.split(',');
  
  // Create purchase record for each app
  for (const appId of appIds) {
    await createPurchase({
      userId: metadata.user_id,
      appId,
      bundleId: metadata.bundle_id,
      amountCents: 0, // Individual price tracked on bundle
      status: 'completed',
    });
  }
  
  // Record bundle purchase
  await createBundlePurchase({
    userId: metadata.user_id,
    bundleId: metadata.bundle_id,
    amountCents: session.amount_total,
  });
}
```

## Edge Cases
- User already owns some apps: Still charge full bundle price? Or pro-rate?
- Recommendation: Full bundle price (simpler), clearly communicate on page
- User buys bundle, then app removed: They keep access
- Future app added: User automatically gets access (if includes_future_apps)
