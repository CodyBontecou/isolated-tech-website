## Overview
Generate dynamic OG images for app pages showing the app icon, name, and tagline. Improves social sharing appearance.

## Current State
- Static `/og-image.png` used for all pages
- App pages use app icon as OG image (small, not ideal)

## Goals
- Each app page gets unique, branded OG image
- Consistent visual style with site branding
- Generated on-demand or at build time
- Fast response times (edge caching)

- Dark background matching site

- Space Mono font for text
- Subtle noise texture overlay

## Implementation Options

### Option 1: Satori + Resvg (Recommended)
Vercel's approach - React components to SVG to PNG.
```typescript
import satori from 'satori';
import { Resvg } from '@resvg/resvg-js';

// Works in Cloudflare Workers with WASM
```

### Option 2: Cloudflare Images
Use Cloudflare's image transformation API with templates.

### Option 3: Pre-generate on app publish
Generate OG images when admin publishes/updates app, store in R2.


## Implementation Sketch
```typescript
// app/api/og/[slug]/route.ts
import satori from 'satori';
import { Resvg } from '@resvg/resvg-wasm';

export async function GET(
  request: Request,
  { params }: { params: { slug: string } }
) {
  const app = await getAppBySlug(params.slug);
  if (!app) return new Response('Not found', { status: 404 });

  const svg = await satori(
    <OGImage name={app.name} tagline={app.tagline} iconUrl={app.icon_url} />,
    { width: 1200, height: 630, fonts: [...] }
  );

  const resvg = new Resvg(svg);
  const png = resvg.render().asPng();

  return new Response(png, {
    headers: {
      'Content-Type': 'image/png',
      'Cache-Control': 'public, max-age=86400', // 24 hours
    },
  });
}
```

## Update Metadata
```typescript
// app/apps/[slug]/page.tsx
export async function generateMetadata({ params }) {
  return {
    openGraph: {
      images: [{
        url: `/api/og/${params.slug}`,
        width: 1200,
        height: 630,
      }],
    },
  };
}
```
