# Visual Regression Testing

## Purpose
Enable safe CSS refactoring by catching visual regressions automatically.

## Approach
Use Playwright's built-in `toHaveScreenshot()` for pixel-perfect comparison.

## Page Groups to Test

### Public Pages (unauthenticated)
- Homepage (`/`)
- App listing (`/apps`)
- App detail (`/apps/[slug]`)
- App changelog (`/apps/[slug]/changelog`)
- App docs (`/apps/[slug]/docs`)
- App FAQ (`/apps/[slug]/faq`)
- Ship Log (`/changelog`)
- Feedback (`/feedback`)
- Roadmap (`/roadmap`)
- Help center (`/help`)
- Privacy (`/privacy`)
- Terms (`/terms`)

### Auth Pages
- Login (`/auth/login`)
- Verify (`/auth/verify`)

### Dashboard Pages (authenticated)
- Dashboard home (`/dashboard`)
- Settings (`/dashboard/settings`)
- Reviews list (`/dashboard/reviews`)

### Admin Pages (admin authenticated)
- Admin home (`/admin`)
- Apps management (`/admin/apps`)
- Users (`/admin/users`)
- Purchases (`/admin/purchases`)
- Codes (`/admin/codes`)

### Viewports
- Desktop (1280x720)
- Mobile (375x667 - iPhone SE)

## Configuration Changes
- Update `playwright.config.ts` with snapshot settings
- Add `.gitattributes` for PNG handling
- Create test helpers for auth state

## Baseline Workflow
1. Run tests once to generate baselines
2. Commit baseline screenshots
3. Future runs compare against baselines
